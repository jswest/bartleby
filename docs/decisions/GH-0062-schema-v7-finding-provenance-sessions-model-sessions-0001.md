# Schema v7 — finding provenance (`sessions.model` / `sessions.harness`, issue #62)

> Source: [#62](https://github.com/jswest/bartleby/issues/62)

findings were attributed to a session but carried no record of *which backend* produced them, which defeats the corpus's purpose (multi-model/multi-harness report evaluation — Claude Code vs local Ollama vs Goose/Pi — stored side-by-side in `findings`). Two nullable columns land on `sessions`, not `findings`: a session is "one run under one backend," every finding it authors shares that backend, and the finding-returning scripts already join `sessions` for `session_name`, so the surfacing is additive (a new `session_provenance` helper in `_common.py` feeds `save_finding`/`edit_finding`/`read_finding`; `list_findings` adds two columns to its existing join). **Attribution is session-level, deliberately not per-finding** — per-finding columns would be redundant denormalization and push the burden onto every `save_finding` call. **Population, ranked by trust**: explicit `bartleby session start --harness --model` (authoritative); best-effort `detect_harness()` reading a small env-signature registry (`CLAUDECODE` → `claude-code` for now, more added as confirmed) when `--harness` is omitted, explicit always overriding; and `bartleby session set` to stamp/correct after the fact (supports the *blind*-comparison workflow: start unset, reveal the backend after assessing). Model rarely has an env signal, so it's explicit/post-hoc only. **NULL = unknown, never fabricated** — an unrecognized harness yields null, not a wrong label (same honesty discipline as `authored_date`). *Asking the model to self-report its own model/harness was rejected as a source of truth* (models are unreliable about their own id and blind to the wrapping harness). Shipped additive via `_upgrade_v6_to_v7` (`ALTER TABLE sessions ADD COLUMN ...`), runnable through `bartleby project upgrade` with no re-ingest; the `test_upgrade_chain_walks_*` gate extended to v7. Surfaced in each script's `--help` `Output:` block and SKILL.md per the #47 contract.
