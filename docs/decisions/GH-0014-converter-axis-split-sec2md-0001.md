# Converter axis split + sec2md (issue #14)

> Source: [#14](https://github.com/jswest/bartleby/issues/14)

the single `backend` config key and `--backend` CLI flag are split into orthogonal axes — `pdf_converter` (`pdfplumber` / `docling`) and `html_converter` (`docling` / `sec2md`). `sec2md` is a third converter slotted into the HTML branch for iXBRL EDGAR filings; it activates only when `html_converter=sec2md` AND a per-file iXBRL sniff (`xmlns:ix="http://www.xbrl.org/.../inlineXBRL"` in the first ~4KB) hits. Non-iXBRL HTML falls through to docling. Hard rename per the no-backwards-compat rule — users re-run `bartleby config`. No schema change: sec2md chunks land under `source_kind='document'` with new `content_type` values `'sec_text'` / `'sec_table'` (split on `Chunk.has_table`). `section_heading` maps to `Chunk.header` directly (often `None` on 8-Ks, populated when sec2md detects section context). XBRL tags, element IDs, and filing-level metadata (CIK / ticker / form type / filing date) are deliberately deferred — they want a schema bump and belong to #10. SGML `.txt` full-submission wrappers were originally out of scope here; they are now unwrapped and routed by a dedicated parser (see the #51 entry below). New optional dep: `bartleby[sec2md]`, pinned `sec2md==0.1.22` (solo dev, alpha, no tagged releases — explicit upgrade control).
