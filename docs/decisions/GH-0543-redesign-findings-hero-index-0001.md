# The /findings list is a hero band over a Table/Grouped index.

The single mint-card column was an underwhelming, hard-to-parse wall: 35 identical-weight cards stacked in one narrow reading column, wasting screen width and surfacing none of the structure the data carries (citation counts span 0–65; findings cluster by research session). The redesign is **spotlight over index**: a constant band of the two **most-recent** findings as large editorial cards, above an index that toggles between a dense scannable table (default) and the same findings grouped by session — client-side toggle state, no persistence. Heroes are most-recent rather than most-cited or user-pinned: most-recent answers "what did I just produce", needs no schema and no pin UI, and self-corrects as new work lands (the tradeoff — a thin newest finding can sit beside a rich one — is accepted and handled by equal-height cards). The hero findings **deliberately reappear** in the index below (in the table and inside their session group): a spotlight over a *complete* list, not a de-dupe bug. Citation count becomes a visual weight cue — a proportional amber bar scaled to the max count on the page, with a `min-width` sliver so a low non-zero count stays visible and a muted/empty state at zero; the bar fill must be a **block** element because an inline span ignores `width` and renders every bar at 0px. The change is **frontend-only**: `listFindings()` already returns `created_at`-DESC rows carrying `session_name` and `citation_count`, so there is no query, server, or schema change. Styles are component-scoped in `findings/+page.svelte` rather than promoted to `app.css`; the `/documents` and `/search` lists share the old `.entity-list` idiom but aligning them is left to a later pass. Rejected: **most-cited heroes** (conflates report size with importance — a tight 5-citation scoop loses to a sprawling 65-citation omnibus), **user-pinned/starred heroes** (the "right" long-term answer but it needs an additive `findings` flag plus a pin control — out of scope for a layout change), and **grouped-by-session as the default index** (session is an artifact of *how* the research happened, not how you want to browse results — better as an opt-in lens than the baseline). No `SCHEMA_VERSION` bump (issue #543).
