# EDGAR full-submission `.txt` unwrap (issue #51, Phase 1)

> Source: [#51](https://github.com/jswest/bartleby/issues/51)

an EDGAR full submission is an SGML envelope (`<SEC-DOCUMENT>` / `<SEC-HEADER>` wrapping N `<DOCUMENT>` blocks). Left to the generic dispatch it fell to the plain character chunker and the agent got raw iXBRL/HTML tag soup with tables shattered cell-by-cell. New `bartleby/ingest/edgar.py` `detect()`/`parse()`/`classify()` splits the envelope and routes each inner document: HTML/iXBRL bodies → sec2md (via a new `sec2md.convert_bytes(html: bytes)` seam, since inner bodies live in memory after the split), plain-text exhibits → the character chunker, graphics (uuencoded / `<TYPE>GRAPHIC`) and XBRL data files (`.xml`/`.xsd`, `EX-101.*`) → skipped with a stderr note. `detect()` is **content-based and extension-independent** (sniffs the envelope markers in a head window), so a `.txt` is caught before the final `else`; the branch sits ahead of the existing `.htm` sec2md branch in `_process_one`. The unwrap also dissolves the issue's secondary "4 KB iXBRL sniff window misses the deep marker" bug for free — each inner body is sniffed from its own byte 0. **One-document model**: the whole submission lands as a single `documents` row (file_hash / archive / one-summary invariants untouched); inner-doc origin is recorded in `section_heading` (sec2md's header, falling back to the inner `<TYPE>`), and chunk indices run continuously across inner docs. **Routing principle — require a converter only when it's the *sole* path to usable output**: inside a submission, inner HTML always goes to sec2md regardless of `html_converter` (the alternative is tag soup, not docling — no working fallback, so the dependency is hard; missing `[sec2md]` raises the existing install hint). This is deliberately asymmetric with the standalone `.htm` path, which keeps honoring `html_converter` and the per-file iXBRL sniff because docling *is* a working fallback there. No schema change (`SCHEMA_VERSION` stays 6); `<SEC-HEADER>` metadata capture (CIK/ticker/form type/date) and the search filter layer remain Phase 2 / #10.
