# Two-tone-by-meaning surface system — the visual unify pass (issue #127)

> Source: [#127](https://github.com/jswest/bartleby/issues/127)

#122 built the token/primitive *plumbing* but deliberately left the *look* alone, so the UI still read as three apps — a polished warm home page, a wall of fully-saturated mint `.surface` cards on every list/detail page, and a third plain-white treatment for search results — with amber (`--color-token-dark`) overloaded as chip *and* link *and* citation color (amber-as-text fails contrast on white). This pass spends the #122 capital on appearance; `web/`-only, no Python/schema/behavior change. **The organizing idea: color is a signal, not decoration.** `.surface` is now the *warm* canonical card (source material the agent works over — documents, search hits, tags, the home dashboard ground); `.surface--finding` is the *mint* variant (the agent's own output — saved findings). The two are **identical but for hue** (same `--radius-lg`, padding, resting `--shadow-card`) precisely so the colour reads as meaning rather than drift; a reader learns the split on the home page (mint Findings card, warm Documents card) and it holds on `/findings` + finding-detail (mint) vs `/documents` + doc-detail + `/search` + `/tags` (warm). **Three supporting moves.** (1) **One surface, two hued variants + one interactive cue**: `.surface--interactive` (the shared border-brighten + shadow-lift hover) goes on cards that lead somewhere (list rows, home cards) and is omitted on static panels (the detail-page report, which is a page not a link); the home cards stopped re-declaring their own box and now *compose* the primitive (`class="surface surface--interactive"`). The faint resting `--shadow-card` is what gives a list figure/ground — it fixes the flat "wall of mint" without changing the palette. (2) **Contrast pass — split amber's jobs**: amber stays for *fills/borders/data* (tag chips, the cite-chip, histogram bars, Doto numerals) but is banished from text; two new tokens carry the readable cases — `--color-link` (dark sage) for navigation (markdown links, home list links, pagination, result titles) and `--color-token-text` (dark amber) for amber-*as-text* accents (`.code`, `.note`, the result `.page-hint`, search's findings-excluded note). (3) **Detail pages made symmetric**: document-detail's prose moved from bare-on-white into a warm `.surface`, mirroring finding-detail's mint panel — same two-pane object in its two colours. **Card radius unified** on `--radius-lg` (the home card's, the polished reference), retiring the `--radius-md` boxes. Sage-as-muted-text (`.meta`/`.summary` at small sizes) was left as-is — it's the established secondary-text language, not an amber-contrast offender, and recoloring it was out of scope. Verified by clean `npm run build` (no unused-selector warnings) and headless before/after screenshots of all eight routes against a live corpus. No schema change.
