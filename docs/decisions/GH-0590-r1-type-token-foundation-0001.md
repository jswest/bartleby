# Self-hosted retro fonts: Iosevka Term + Source Serif 4 + Doto (issue #590)

> Source: [#590](https://github.com/jswest/bartleby/issues/590)

The R1 retro foundation drops the Google Fonts `<link>` from `app.html` and self-hosts all three type voices as woff2 under `bartleby/web/static/fonts/`, declared via `@font-face` in a new `src/fonts.css` that `app.css` `@import`s. Self-hosting (not the CDN) is forced by two hard requirements the CDN can't meet: **(1)** Google's served Source Serif 4 subset strips the `onum` (oldstyle-figures) OpenType feature, which the locked design needs on prose numerals — the full Adobe variable font carries it (verified by loading the woff2 with fontTools and confirming `onum` in the GSUB FeatureList; present in both upright and italic); and **(2)** "Iosevka Term" specifically (the terminal/UI variant, narrower than plain Iosevka) isn't on a clean CDN. All three families are **SIL Open Font License 1.1**, which expressly permits web embedding and self-hosting, so committing the `.woff2` files to the repo (normal practice for self-hosted webfonts) is licit. Build pipeline: **Iosevka Term** ships as three static weights (Regular/Medium/Bold) extracted face-0 from the local `.ttc` TrueType *collections* (`fontTools.ttLib.TTFont(path, fontNumber=0)` — face 0 is "Iosevka Term" proper, not the Extended/Oblique siblings in the same collection), then subset via `fontTools.subset` to a Latin + punctuation + arrows/daggers codepoint set (`U+0000–017F`, general punctuation, currency, `→ ‹ †‡ — …`) and flavored to woff2 (~114 KB/face; the full unsubset face is ~1.5 MB and Iosevka's whole nine-weight family is unnecessary for chrome). The first subset attempt used `Subsetter` with `desubroutinize=True` and a mis-built populate set; that produced a malformed `glyf` table Chrome's OTS rejected ("Bad glyph flag … bit 6 must be set to zero") so the faces silently fell back to system mono — caught via the browser console, not the network panel (the files 200'd fine). Re-subsetting through `fontTools.subset.main` with `--no-hinting --notdef-outline` and no desubroutinize produced clean, OTS-valid faces (confirmed: zero decode warnings, `document.fonts` reports `loaded`, canvas measurement shows real Iosevka metrics, not fallback). **Source Serif 4** and **Doto** stay full variable fonts (single file spans the weight axis) compressed straight through `woff2_compress` — Source Serif needs broad prose coverage and carries `onum`; Doto is tiny (~9 KB) and display-only. The tokens: `--font-display` = Doto (display only, gated by a new `--text-display-floor: 1.25rem` so the dot-matrix never renders muddy small), `--font-sans` = Iosevka Term (the chrome/UI role — name kept), `--font-serif` = Source Serif 4 (prose, with `font-feature-settings: "onum" 1` on `li,p`/`.desc`/`.markdown-body`; tables pin `lnum` so columnar figures stay aligned), `--font-mono` = Iosevka Term. Self-hosting is also why `pyproject.toml`'s `[tool.setuptools.package-data]` gains `web/static/**/*` — per the #87 packaging invariant, an installed wheel must ship the fonts, and `serve.py`'s `_sync_web` already copies/symlinks every non-preserved entry under `web/` (so `static/` rides along with no code change). No schema change.
