# Web style standardization — design tokens + extracted primitives (issue #122)

> Source: [#122](https://github.com/jswest/bartleby/issues/122)

the SvelteKit UI's styling foundation was sound (CSS-var tokens in `app.css`, shared `.markdown-body`/`.split`/`.entity-list`) but had drifted — the same visual concept existed in 2–4 slightly-different copies (tag chip styled two ways, the error box copy-pasted across three routes, the submit button defined three times, the filter-form chrome duplicated between `/search` and `/documents`, and `SearchResult`/`ScanMatch` ~90% identical). Consolidated, `web/`-only, no Python/schema change. **Three move types, by altitude.** (1) **Tokens** — `app.css:root` gained a full scale: text/surface/danger colors, a `--space-*` ramp, a `--text-*` ramp, `--radius-*`, `--tracking-wide`, and one `--width-content`. Every route `<style>` was converted; the only raw `rem` left are *structural dimensions* (input widths, grid track sizes, fixed heights, the deliberately-wider 52rem home dashboard) — that boundary (rhythm tokenized, dimensions literal) is the rule for future work. (2) **Components — only for markup+behavior+variants**: `Button.svelte` (md/sm), `StatusBanner.svelte` (error/busy), and `ResultCard.svelte` (search/scan, merging the two deleted result components). (3) **Utility classes — for presentation-only primitives**: `.surface` (the off-light card, now *composed* via `class="entity surface"` on the document/finding/tag list items rather than re-declared), the unified `.tag` chip, the shared filter-form chrome (`.filters`/`fieldset`/`legend`/`.checks`/inputs), and the text atoms (`.meta`/`.empty`/`.summary`/`.note`/`.muted`/`.code`/`.desc`). **Deliberately not components** (the one judgment call): Surface/Chip/form-chrome are classes, not `.svelte` files — a component there adds only import noise since there's no markup or behavior to encapsulate. Net −254 lines; verified by `npm run build` (clean, no unused-selector warnings) and SSR-rendering every route against a live corpus. No schema change.
