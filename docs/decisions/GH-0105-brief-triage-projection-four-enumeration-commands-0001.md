# `--brief` triage projection on the four enumeration commands (issue #105)

> Source: [#105](https://github.com/jswest/bartleby/issues/105)

triage over a large corpus means many list/search calls, and the high-token enumeration commands returned more per row than triage needs (an agent on a 707-doc corpus hit the output cap on `list_documents --limit 200` and resorted to parsing the spilled JSON by hand). The fix **standardizes the vocabulary, not the behavior**: a `--brief` flag on `search`, `list_documents`, `scan`, and `list_findings`, each documenting *its own* skinniest-useful per-row projection — `search` → `chunk_id`/`source_kind`/`source_name`/`page_number`/`rank`/`normalized_score` + a truncated `text` preview (drops the full text, context arrays, and image locators; `--add-context` is ignored under `--brief`); `list_documents` → `id`/`file_name`/`title`; `scan` → locators only (`document_id`/`file_name`/`chunk_id`/`page_number`, dropping the snippet + `text_length`, so `--preview` is moot); `list_findings` → `finding_id`/`title`/`citation_count`. **One flag name, per-command semantics — deliberately not one shared projection helper** (the field sets have nothing in common; a shared abstraction would be indirection for its own sake). The **envelope is invariant**: pagination (`total`/`offset`/`limit`/`hint`) and status flags (`memory_excluded`/`excluded_null_dated`) are always present, and `--brief` adds **no** echo field — default output shapes stay byte-for-byte unchanged (so brief is purely additive, no compat shim needed). On `list_documents`, `--brief` is a third tier *below* the existing default, mutually exclusive with `--verbose` via an argparse `add_mutually_exclusive_group()` (exits 2 like other arg-level errors); `--verbose` stays the only "more columns" knob since it's the only command with extra columns. **Out of scope**: `read_chunks`/`read_document`/`read_finding` (full-content-by-contract — a brief tier is meaningless or contract-breaking there), and a generic `--fields a,b,c` selector (canned `--brief` is a simpler, more predictable contract for an agent; revisit only if the canned projections prove too rigid). The per-script loop shape diverges on purpose: `search`/`list_documents` use a brief-branch-with-`continue` because their loops do per-row work brief skips (context fetch / image locators; `--verbose` extension), while `scan`/`list_findings` use two list comprehensions because their loops *only* project. No schema change.
