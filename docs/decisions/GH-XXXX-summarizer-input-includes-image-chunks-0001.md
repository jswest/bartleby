# Summarizer input includes image chunks

ingest-time summarization feeds the LLM document chunks *and* image chunks (the VLM's `image_description` or Tesseract's `image_ocr`) interleaved by `(page_number, chunk_index)` rather than the raw extracted body alone. Image-heavy docs (slide decks, figure-laden papers, OCR-fallback pages) get the VLM's already-paid-for analysis folded into the summary input. Implemented in `_build_summary_input` in `bartleby/commands/scribe.py`: it builds from whatever chunks exist (document, image, or both), so an image-only doc summarizes from its real VLM/Tesseract descriptions rather than the raw `full_text`. The `full_text` fallback is reached only when a doc has *zero* chunks — and `_maybe_summarize` excludes that case outright (see the #80 entry). Decorative-image bloat is absorbed by the `max_summarize_tokens` cap (default 50k).
