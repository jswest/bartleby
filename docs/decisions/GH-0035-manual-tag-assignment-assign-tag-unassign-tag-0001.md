# Manual tag assignment — `assign_tag` / `unassign_tag` (issue #35)

> Source: [#35](https://github.com/jswest/bartleby/issues/35)

two skill scripts that attach/detach a single `(document, tag)` pair directly, with no LLM. The `tag` classifier only reads document *summaries*, so body-level/intrinsic categories it structurally cannot see — OCR quality, language, "contains tables" — are unreachable through it (summaries are reconstructed clean prose; the defining signal is scrubbed out). These primitives record membership the agent/human determined out-of-band (body scan, deterministic rule), which is the only path for such tags. They reuse the existing `assign()` / `unassign()` helpers in `_tags.py`; `assign_tag` is idempotent (`INSERT OR IGNORE`), `unassign_tag` is a no-op when absent and — unlike `delete_tag` — detaches one document rather than cascading the whole tag. No DB or schema change. Deliberately not built: a provenance column distinguishing manual from LLM assignment (assignments are assignments), a body-aware classification mode for `tag` (deferred — an LLM is the wrong tool for the motivating `bad_ocr` case anyway), and multi-*tag* forms (still one tag per call). The "single doc; agents loop" stance was reversed by issue #68 (see entry below) once an agent hit 63 process spawns on one tagging pass — the multi-*document* batch form is now the only form.
