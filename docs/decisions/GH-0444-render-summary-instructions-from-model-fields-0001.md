# Render summarizer instructions from DocumentSummary.model_fields (issue #444)

> Source: [#444](https://github.com/jswest/bartleby/issues/444)

The summarizer prompt no longer carries a hand-maintained restatement of the `DocumentSummary` field definitions; `SUMMARY_INSTRUCTIONS` (in `bartleby/providers/prompt.py`) is now built at import time from a one-line framing sentence plus a per-field bullet rendered from `DocumentSummary.model_fields[name].description`. Those Field descriptions in `bartleby/providers/base.py` were already the enforced source of truth for every provider's structured-output channel (Anthropic tool-use `input_schema`, OpenAI `response_format` json_schema, Ollama `format=`), so this collapses two hand-synced copies into one.

The duplication had already drifted exactly as the benchmark judge module warns against (`bartleby/benchmark/judging.py` keeps `JUDGE_INSTRUCTIONS` framing-only precisely so the `JudgeScore` thresholds can't fork): the old prompt's `description` bullet silently lost the `never exceed 200 characters` ceiling that both the schema description and the judge rubric impose. **What is no longer handled:** the ability for the prompt text and the schema descriptions to say different things. That ability was only ever a drift hazard, and it had already been realized, so nothing of value is lost.

**No model sees less instruction than before â€” and one provider channel was load-bearing here.** Every real provider (`anthropic`, `openai`, `ollama`) routes through `build_summary_messages`, which interpolates `SUMMARY_INSTRUCTIONS`; `wsjpt.py` consumes the constant directly as `custom_instructions` without ever calling `build_summary_messages`. So the field rules had to be rendered **into the constant**, not only into the message builder â€” otherwise the wsjpt/Gemini channel would have silently lost all four field bullets. Likewise, Ollama's `format=` is grammar-constrained decoding: the schema descriptions never reach that model as prompt text, so the rendered content in the prompt is what carries the rules there. Both constraints are why the constant keeps the full per-field content rather than shrinking to a `JUDGE_INSTRUCTIONS`-style framing line.

`summarize.prompt_sha()` (`bartleby/benchmark/summarize.py`) hashes `build_summary_messages("{{DOCUMENT}}")` together with the schema, so it takes a one-time provenance break at this switch â€” exactly the kind of regime change the leaderboard's heterogeneity warning was built to surface on stores that mix old and new `prompt_sha`. No test asserted the old prompt text, so none needed rewriting. No schema change (`SCHEMA_VERSION` stays 9).
