# Content-sniffed file-type resolution (issue #78)

> Source: [#78](https://github.com/jswest/bartleby/issues/78)

ingest type detection was purely extension-based (`path.suffix.lower()` against `SUPPORTED_EXTENSIONS`), so a valid file with a missing/mangled extension was rejected (single-file) or **silently dropped** (directory `rglob` filter) — the motivating case was an NTSB docket of 16 PDFs whose scraper truncated long filenames, lopping the `.PDF` off 7 of them (6 left with empty `suffix`, 1 with junk `.1_ january` because the last dot fell inside `5800.1`). New `resolve_extension(path)` in `bartleby/ingest/chunk.py` is the single resolution seam: a **recognized extension is trusted as-is and never second-guessed from content** (a `.txt` holding PDF bytes stays text — pinned by `test_resolve_extension_does_not_override_recognized_extension`); only a missing/unrecognized extension is resolved by sniffing magic bytes via the `filetype` library (pure-Python, no system dep), mapping back to a `SUPPORTED_EXTENSIONS` member or `None`. **Sniff only when info is missing, never as a fallback to override a recognized-but-"wrong" extension** — that's the deliberate asymmetry the user asked for. `_collect_files` now returns `(sources, unidentified)` where `sources` are `(Path, resolved_ext)` pairs; the resolved ext is threaded through `_archive` (so the archived copy is named correctly) and `_process_one`'s dispatch (replacing the old in-line `path.suffix`). **Unsupported files are still skipped, but no longer silently** — directory ingest surfaces a capped list of files it couldn't identify (single-file still raises `ValueError`). This composes with the EDGAR `detect()` path below: a recognized `.txt` resolves to `.txt`, then content-routing inside `_process_one` still catches an SGML envelope. No schema change. New runtime dep `filetype>=1.2.0`. **Tangent folded in**: the `wsjpt` optional extra (git source on WSJ-internal `github.dowjones.net`) was removed from the locked dependency set — it is unreachable off-VPN and broke `uv lock`/`uv sync` for everyone; it is now an out-of-band `uv pip install 'git+ssh://…/wsjpt.git'` (the provider already imports `wsjpt` lazily with an install-hint `RuntimeError`, so nothing else changes).
