# The anthropic-cc judge grades via the local `claude -p` CLI under the user's subscription.

`bartleby benchmark judge --model anthropic-cc/<model>` runs the judge through the locally-installed Claude Code CLI (`claude -p --output-format json --json-schema …`) authenticated by the user's own Claude subscription, as an alternative to the OpenAI judge that needs `OPENAI_API_KEY`. We chose the headless CLI over the Claude Agent SDK and over a raw Anthropic API call for three reasons: the `claude` binary ships with Claude Code, so there is no new dependency; running it as a subprocess gives explicit control of the child environment, which is the load-bearing billing-safety mechanism — Claude Code's auth precedence puts `ANTHROPIC_API_KEY`/`ANTHROPIC_AUTH_TOKEN` *above* the subscription OAuth login, so `cc_judge` `env.pop`s both (otherwise a judge run silently bills a pay-as-you-go API account at API rates) and never passes `--bare` (which forces API-key auth and never reads OAuth/keychain); and `--json-schema` gives the same validate-and-retry structured-output guarantee the SDK would, with the 1–5 rubric axes expressed as `enum` (Claude's structured outputs don't enforce numeric `minimum`/`maximum`) and `JudgeScore.model_validate` as the final Pydantic check. Legitimacy was confirmed against Anthropic's own docs (the [Agent SDK + Claude plan support article](https://support.claude.com/en/articles/15036540-use-the-claude-agent-sdk-with-your-claude-plan) and the [Claude Code legal & compliance page](https://code.claude.com/docs/en/legal-and-compliance)): subscription-authenticated Agent SDK / `claude -p` use is sanctioned for *ordinary, individual* use (own login, own machine) and explicitly covers "third-party apps that authenticate with your Claude subscription"; it is *not* for shared/multi-tenant service use, which must use a Console API key. `anthropic-cc` is registered as a `LOCAL_PROVIDER` (its wall-clock is machine-local) but is judge-only today — `summarize` rejects it. Any CLI failure (no login, non-zero exit, retry-exhausted, schema mismatch) becomes an `ok: False` record like the OpenAI judge, never a crash or silent spend. Login is preflighted with `claude auth status` (no billed turn), and the judge runs with `--tools ""` so it stays a pure completion (issue #295).
